# 006 组合适配器管线重构

时间：2026-04-27

## 目标

实现“默认规则 + 站点适配器”组合生效机制：

- 默认规则永远执行
- 命中站点后叠加对应站点适配器
- 通过优先级控制覆盖顺序，而不是二选一

## 主要改动

1. 适配器解析从“单个命中”改为“命中集合”
   - 新增 `resolveAdapters()` 返回当前 URL 命中的适配器数组
   - 按 `priority` 升序执行（默认先跑，站点后跑）
2. 注册模型增强
   - `registerAdapter` 支持 `priority`
   - `default` 适配器设置 `priority: 0`
   - `bilibili` 适配器设置 `priority: 100`
3. 执行模型重构
   - 新增 `runAdapters()`，依次执行每个适配器的选择器逻辑与 `afterApply`
   - `MutationObserver` 触发时也走同样的组合管线
4. 规则职责调整
   - 根样式 `cssForRoot` 仅保留全站通用基础能力
   - B站兜底增强迁移到 `bilibili.afterApply`，作为站点专属补丁

## 结果

符合“组合生效”的设计：

- 一个网站会先应用 `default`，再应用其站点适配器
- 站点适配器只负责修复默认规则覆盖不到或表现不佳的区域
- 不同网站适配器彼此独立，后续可直接新增
