## 052 — 个人中心：投稿主题展示与「已保存方案」文案

### 变更说明

1. **新增接口** `GET api/entry/user/themes/submissions`：返回当前用户作为 `submitterId` 提交的全部主题记录（含 `isActive`，未审核通过的也会返回），用于个人中心汇总展示。
2. **`home/app/pages/account.vue`**
   - 当有投稿时在页面上方展示 **「我提交的主题」** 区块（高亮边框与背景），每张卡片附带 **待审核 / 已上线** 说明；待审核条目使用 `ThemeCard` 的 `asPreview`，避免误触发「设为账号主题」等与未上架逻辑不一致的操作。
   - 原先用 `account.title`（「我的账号」）充当「当前主题」小标题的问题已修正：改为 **「已保存的亮色 / 暗色方案」**，并增加简短说明，明确数据来自用户表中的 `lightTheme` / `darkTheme` 字段、用于 **网站与扩展同步**。
   - 亮色/暗色主文案优先展示主题的 **显示名称（`displayName`）**，通过收藏列表 + 上架主题列表 + 投稿列表做一次 slug 反向解析；**技术标识**单独以小字单行展示。
   - 收藏区分标题由原来的「亮色主题」（易与上文混淆）改为 **「收藏的亮色主题 / 收藏的暗色主题」**。
3. **文案**：`zh.json` / `en.json` 中 `account` 命名空间增补上述条目。

### 原因

- 用户投稿需要在个人中心可追溯、与「商城收藏」区分。
- 个人中心先前重复「我的账号」标题且展示内部 `name` slug，难以理解用途；与产品预期（人类可读名称 + 说明同步用途）不一致。
