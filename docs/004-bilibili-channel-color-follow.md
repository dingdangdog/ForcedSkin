# 004 B站频道区跟随主题修复

时间：2026-04-27

## 问题背景

Bilibili 顶部频道区（`bili-header__channel`）出现“部分文字和图标不跟随主题”的现象，典型表现为链接文本和 SVG 图标保持站点默认灰色。

## 根因分析

1. 该区域大量图标颜色通过 `svg path` 的 `fill` 属性直接写死（例如 `#61666Dff`）
2. 原有策略主要改普通文本与容器颜色，对内联 `path fill` 处理不足
3. `data-v-*` 为前端框架（Vue）生成的 scoped 标记，不是主题失效的根因

## 本次修复

1. 扩充 B 站适配器选择器
   - 覆盖 `bili-header__channel`、左右频道容器、频道链接等节点
2. 频道区文本强制跟随主题
   - 频道主链接/右侧链接使用前景色
   - 图标标题（如“动态”）使用主题主色
3. 单色 SVG 图标统一主题化
   - 针对频道区 `path`，对可主题化的单色 fill（hex/rgb/currentColor）改为 `currentColor`
   - 继承所属链接颜色，保证图标与文字风格一致
4. 保留风险控制
   - 仍跳过图片、视频、蒙版、复杂混合模式等高风险场景，避免“蒙版看不见”问题复发

## 结论

`data-v-*` 不会天然导致“不跟随主题”。只要不依赖该属性做匹配，而是基于稳定 class 与结构选择器，就可以稳定适配并持续扩展。
