# 003 适配器路由重构记录

时间：2026-04-27

## 变更目标

将主题应用逻辑重构为“适配器路由 + 通用智能引擎”架构，支持按网址匹配不同适配器，并可持续扩展更多特殊站点。

## 本次变更内容

1. 重构 `src/content/content.js` 为可扩展架构
   - 新增适配器注册机制：`registerAdapter(adapter)`
   - 新增路由匹配机制：`resolveAdapter()` 按 `hostname/url` 选择适配器
   - 默认适配器兜底，未命中特殊站点时仍可应用基础主题
2. 引入通用智能改色引擎
   - 采用“背景/文字/边框”三类选择器分层处理
   - 仅对候选元素定向处理，避免全量覆盖
   - 对媒体元素、复杂背景、蒙版、混合模式、backdrop-filter 等高风险节点自动跳过
   - 文本改色加入对比度判断（低对比场景才强制修正）
3. 新增 Bilibili 适配器
   - 通过域名路由：`bilibili.com` 和 `*.bilibili.com`
   - 针对 B 站常见页面结构设置专用选择器
   - 额外处理作者/统计等次级文本色，提升信息层级清晰度
4. 动态页面增量适配
   - 使用 `MutationObserver` 监听页面内容变化
   - 通过节流重新执行适配，提升 SPA 场景稳定性

## 扩展方式说明

后续新增网站适配器时，只需继续调用 `registerAdapter({...})` 增加一段配置：

- `id`: 适配器唯一标识
- `match(hostname, url)`: 路由匹配逻辑
- `selectors`: `background` / `border` / `text` 三类选择器
- `afterApply`（可选）: 站点专项细化处理

这样可以在不破坏通用逻辑的前提下，按站点逐步提升效果。
